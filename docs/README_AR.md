# 📱 توثيق تطبيق أندرويد: TwinklingStars

---

## 🧾 معلومات عامة
**اسم المشروع:** TwinklingStars  
**المؤلف(ون):** Zeev Fraiman  
**التاريخ:** مايو 2024  
**اللغة:** Java  
**بيئة التطوير:** Android Studio  
**إصدار أندرويد:** minSdk 28 / targetSdk 36  

---

## 🎯 هدف المشروع
*   **المهمة التي يؤديها التطبيق:** يوفر خلفية متحركة ديناميكية وجذابة بصرياً لنجوم متلألئة.
*   **الأهمية:** يعزز جمالية واجهة المستخدم ويعتبر مثالاً تعليمياً لإنشاء رسوم متحركة مخصصة (Custom View) باستخدام Canvas API في أندرويد.
*   **الجمهور المستهدف:** مطورو أندرويد المهتمون بالأمثلة الرسومية والمستخدمون الذين يفضلون الواجهات الجمالية.

---

## 📌 متطلبات التطبيق

### المتطلبات الوظيفية
*   توليد 150 نجمة فريدة بمواقع وأحجام وأعداد رؤوس (5 أو 6) عشوائية.
*   تحريك تأثير التلألؤ (تغيير الشفافية - Alpha).
*   تحريك تأثير الدوران لكل نجمة.
*   الحفاظ على سلاسة الرسوم المتحركة بمعدل 60 إطاراً في الثانية (FPS).

### المتطلبات غير الوظيفية
*   **الأداء:** رسم محسن باستخدام كائنات قابلة لإعادة الاستخدام لتقليل استهلاك الذاكرة وعمل Garbage Collector.
*   **سهولة الاستخدام:** يعمل التطبيق مباشرة عند التشغيل دون الحاجة لإعدادات.
*   **الموثوقية:** حلقة رسوم متحركة مستقرة دون تسريب للذاكرة.

---

## 🧠 الهيكل العام
*   **النهج المتبع:** Custom View مع فصل النموذج (`Star`).
*   **لماذا تم اختيار هذا النهج:** بالنسبة لمشروع رسوم متحركة متخصص، فإن بنية MVVM/MVP الكاملة تعتبر مبالغاً فيها. استخدام View مخصص يدير قائمة من كائنات النموذج يوفر أفضل أداء وبساطة في الكود.
*   **مكونات النظام الأساسية:**
    *   `MainActivity`: النشاط المستضيف.
    *   `StarfieldView`: محرك الرندرة ومتحكم الرسوم المتحركة.
    *   `Star`: كائن بيانات يحمل حالة ومنطق كل نجمة على حدة.

## 🧩 وصف تفصيلي للكلاسات

### 📌 الكلاس: MainActivity
*   **الدور:** نقطة الدخول.
*   **المسؤولية:** تهيئة النشاط وإعداد واجهة العرض (Layout).
*   **الطرق (Methods) الرئيسية:**
    *   `onCreate()`: طريقة دورة الحياة القياسية لإعداد واجهة المستخدم.
*   **التفاعل:** يقوم بتحميل الـ Layout الذي يحتوي على `StarfieldView`.

### 📌 الكلاس: StarfieldView
*   **الدور:** المحرك الرسومي.
*   **المسؤولية:** توليد النجوم، إدارة حلقة الرسوم المتحركة، ورسم الأشكال الهندسية على الـ Canvas.
*   **الطرق الرئيسية:**
    *   `init()`: إعداد أدوات الرسم (Paint) وتهيئة قائمة النجوم.
    *   `generateStars()`: ملء قائمة النجوم بخصائص عشوائية.
    *   `onDraw(Canvas)`: جوهر الرسوم المتحركة؛ يقوم بتحديث ورسم النجوم.
    *   `createStarPath()`: حساب المسار الهندسي (Path) لشكل النجمة.

### 📌 الكلاس: Star
*   **الدور:** نموذج / كائن بيانات.
*   **المسؤولية:** تخزين الحالة (الموقع، الدوران، الشفافية) وحساب الحالة التالية للرسوم المتحركة.
*   **الطرق الرئيسية:**
    *   `update()`: منطق تغيير الشفافية (التلألؤ) وزاوية الدوران.

## 🔄 مخطط عمل التطبيق
1.  **البدء:** يفتح المستخدم التطبيق.
2.  **التهيئة:** يبدأ `MainActivity` ويقوم `StarfieldView` بإنشاء 150 كائناً من نوع `Star`.
3.  **الحلقة:** يتم استدعاء `onDraw`:
    *   تقوم كل نجمة بتحديث الـ `alpha` والـ `rotation` الخاص بها.
    *   يقوم `StarfieldView` برسم كل نجمة على الـ `Canvas`.
    *   يتم جدولة الإطار التالي باستخدام `postInvalidateDelayed(16)`.
4.  **العرض:** يرى المستخدم رسوماً متحركة سلسة ولانهائية لسماء مرصعة بالنجوم.

## 🎨 تحليل UI/UX
*   **اختيار التصميم:** خلفية زرقاء داكنة عميقة (`#000033`) تحاكي سماء الليل الواقعية.
*   **المبادئ:**
    *   **البساطة:** لا توجد أزرار أو قوائم تشتت الانتباه عن المشهد البصري.
    *   **المنطقية:** النجوم المولدة عشوائياً تحاكي الطبيعة.
    *   **سهولة الوصول:** تباين عالٍ بين النجوم البيضاء والخلفية المظلمة.
*   **التحسينات الممكنة:** إضافة عناصر تفاعلية (نجوم تتفاعل مع اللمس).

---

## ⚙️ إدارة الخيوط (Threads)
*   **الآلية:** استخدام `postInvalidateDelayed` على خيط واجهة المستخدم الرئيسي (UI Thread).
*   **لماذا:** هذه هي الطريقة القياسية والفعالة لتنفيذ الرسوم المتحركة البسيطة المتزامنة مع معدل تحديث الشاشة.
*   **الوقاية:**
    *   **ANR:** جميع الحسابات سريعة جداً (O(n) لـ 150 نجمة).
    *   **تسريب الذاكرة:** لا توجد خيوط خلفية طويلة الأمد أو مراجع ثابتة للسياق (Context).

---

## 💾 التعامل مع البيانات
*   **التخزين:** يتم الاحتفاظ بالبيانات في ذاكرة الوصول العشوائي (RAM) في قائمة من كائنات `Star`.
*   **السبب:** بما أن النجوم تتولد ديناميكياً عند كل تشغيل، فلا داعي للتخزين الدائم (SQLite/Room).
*   **سلامة البيانات:** يتم ضمانها من خلال التغليف داخل كلاس `Star`.
