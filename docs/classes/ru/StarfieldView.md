# Класс: StarfieldView

## 1. Общая информация
*   **Имя класса:** `StarfieldView`
*   **Тип:** Обычный класс (Пользовательский View / Custom View)
*   **Назначение:** Этот класс является «холстом» и «движком» приложения. Он отвечает за отрисовку всего ночного неба и управление анимацией всех 150 звезд. Он производит расчеты для создания формы звезд и обеспечивает их плавное движение.
*   **Взаимодействие:** 
    *   Размещается внутри макета `activity_main.xml`.
    *   Создает и управляет списком объектов `Star`.
    *   Взаимодействует с системой `Canvas` в Android для рисования графики на экране.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `stars` | `List<Star>` | Коллекция (список), содержащая все 150 объектов звезд. | `generateStars()`, `onDraw()` |
| `paint` | `Paint` | «Кисть», используемая для рисования звезд. Определяет цвет и сглаживание. | `init()`, `onDraw()` |
| `random` | `Random` | Инструмент для генерации случайных чисел (позиций, размеров). | `generateStars()` |
| `NUM_STARS` | `int` (static) | Константа, равная 150, определяющая количество создаваемых звезд. | `generateStars()` |
| `starPath` | `Path` | Переиспользуемый объект геометрической формы для линий звезды. | `createStarPath()` |

## 3. Методы класса

### Имя метода: `StarfieldView` (Конструкторы)
*   **Тип:** `public`
*   **Параметры:** `Context context`, `AttributeSet attrs` (необязательно).
*   **Что делает:** Это стандартные конструкторы, необходимые для Android. Они вызывают `init()` для подготовки View.
*   **Когда вызывается:** При запуске приложения и загрузке макета (layout).

### Имя метода: `init`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Что делает:** 
    1.  Устанавливает цвет кисти (`paint`) на белый.
    2.  Включает сглаживание («anti-alias»), чтобы края звезд были плавными, а не пиксельными.
    3.  Вызывает `generateStars()` для создания данных о звездах.
*   **Когда вызывается:** Один раз при создании View.

### Имя метода: `generateStars`
*   **Тип:** `private`
*   **Что делает:** 
    1.  Вычисляет ширину и высоту экрана.
    2.  Запускает цикл 150 раз.
    3.  Каждый раз выбирает случайные значения для позиции, размера, скорости и вращения.
    4.  Создает новый объект `Star` и добавляет его в список `stars`.
*   **Когда вызывается:** Во время инициализации.

### Имя метода: `createStarPath`
*   **Тип:** `private`
*   **Возвращаемое значение:** `Path` (Геометрическая форма звезды).
*   **Параметры:** `cx`, `cy` (центр), `outerRadius` (размер), `points` (5 или 6 лучей), `rotation`.
*   **Что делает:** Это математическая часть. Метод вычисляет координаты X и Y для каждой точки звезды (внешние вершины и внутренние «впадины») и соединяет их линиями, образуя многоугольник.
*   **Когда вызывается:** Каждый раз, когда нужно нарисовать звезду.

### Имя метода: `onDraw`
*   **Тип:** `protected`
*   **Параметры:** `Canvas canvas` (экран, на котором нужно рисовать).
*   **Что делает:** 
    1.  Заполняет фон темно-синим цветом.
    2.  Проходит циклом по всем 150 звездам.
    3.  Для каждой звезды устанавливает прозрачность кисти (`alpha`).
    4.  Рассчитывает текущую форму звезды через `createStarPath`.
    5.  Рисует её на `canvas`.
    6.  Приказывает звезде обновить свое состояние (`update()`).
    7.  **Цикл анимации:** Вызывает `postInvalidateDelayed(16)`, что говорит системе: «Подожди 16 миллисекунд и снова запусти `onDraw`».
*   **Когда вызывается:** Вызывается системой каждый раз, когда View нужно перерисовать (постоянно во время анимации).
*   **Важно:** 16 миллисекунд выбрано потому, что 1000 мс / 60 кадров ≈ 16.6 мс. Это создает плавную анимацию 60 FPS.

## 4. Жизненный цикл
*Не применимо (View, а не Activity).*

## 5. Взаимодействие с интерфейсом (UI)
*   Этот класс *и есть* компонент интерфейса. Он определен в `activity_main.xml` и заполняет весь фон экрана.

## 6. Взаимодействие с другими компонентами
*   **Управление моделью:** Он «владеет» объектами `Star` и управляет их жизненным циклом.

## 7. Общая логика класса
1.  **Подготовка:** Создать 150 звезд со случайными свойствами.
2.  **Отрисовка:** Покрасить фон в синий, затем нарисовать каждую звезду в зависимости от её положения и яркости.
3.  **Итерация:** Немного изменить свойства звезд и попросить систему немедленно перерисовать экран.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте `StarfieldView` как кинопроектор и кинопленку.
- Метод `generateStars` — это съемка сцен.
- Метод `onDraw` — это лампа проектора. Она показывает текущий кадр, затем быстро передвигает «актеров» (звезды) на крошечное расстояние и показывает следующий кадр.
Она делает это так быстро (60 раз в секунду), что ваши глаза видят плавное движение вместо отдельных картинок.
