# Класс: Star

## 1. Общая информация
*   **Имя класса:** `Star`
*   **Тип:** Обычный класс (Модель данных)
*   **Назначение:** Этот класс представляет собой одну звезду в нашем анимированном небе. Он служит «чертежом» или «контейнером», в котором хранится вся информация об одной звезде: где она находится, какого она размера, насколько она яркая и как быстро она вращается.
*   **Взаимодействие:** Используется классом `StarfieldView`. `StarfieldView` создает большой список объектов `Star` и приказывает каждому из них обновить свое состояние перед отрисовкой.

## 2. Переменные (Поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `x` | `float` | Горизонтальное положение звезды на экране. | `Конструктор`, `StarfieldView.createStarPath` |
| `y` | `float` | Вертикальное положение звезды на экране. | `Конструктор`, `StarfieldView.createStarPath` |
| `size` | `float` | Радиус/диаметр звезды. | `Конструктор`, `StarfieldView.createStarPath` |
| `alpha` | `float` | Текущая прозрачность (0.0 = невидима, 1.0 = полностью видна). | `Конструктор`, `update()`, `StarfieldView.onDraw` |
| `twinkleSpeed` | `float` | Как быстро звезда меняет свою яркость. | `Конструктор`, `update()` |
| `maxAlpha` | `float` | Максимальная яркость, которой может достичь звезда. | `Конструктор`, `update()` |
| `minAlpha` | `float` | Минимальная яркость (насколько тусклой она становится). | `Конструктор`, `update()` |
| `rotation` | `float` | Текущий угол поворота звезды в градусах. | `Конструктор`, `update()`, `StarfieldView.createStarPath` |
| `rotationSpeed` | `float` | Как быстро звезда вращается вокруг своей оси. | `Конструктор`, `update()` |
| `points` | `int` | Сколько лучей/вершин у звезды (например, 5 или 6). | `Конструктор`, `StarfieldView.createStarPath` |
| `isRising` | `boolean` | Отслеживает, становится ли звезда ярче или тусклее в данный момент. | `Конструктор`, `update()` |

## 3. Методы класса

### Имя метода: `Star` (Конструктор)
*   **Тип:** `public`
*   **Возвращаемое значение:** Нет (это конструктор, он создает объект).
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `x`, `y` | `float` | Координаты положения. |
    | `size` | `float` | Размер звезды. |
    | `maxAlpha`, `minAlpha`| `float` | Пределы яркости. |
    | `twinkleSpeed` | `float` | Скорость изменения яркости. |
    | `rotation` | `float` | Начальный угол поворота. |
    | `rotationSpeed` | `float` | Скорость вращения. |
    | `points` | `int` | Количество лучей звезды. |
*   **Что делает:** Инициализирует новую звезду с определенными чертами. Устанавливает начальную яркость на минимум и приказывает звезде начать «расти» (становиться ярче).
*   **Когда вызывается:** Когда `StarfieldView` создает звезды при запуске приложения.
*   **Что важно понимать:** Конструктор позволяет нам иметь множество звезд, которые выглядят и ведут себя по-разному.

### Имя метода: `update`
*   **Тип:** `public`
*   **Возвращаемое значение:** `void` (ничего не возвращает).
*   **Параметры:** Нет.
*   **Что делает:** 
    1.  **Мерцание:** Если `isRising` равно true, к `alpha` добавляется `twinkleSpeed`. Если яркость достигает `maxAlpha`, она начинает уменьшаться.
    2.  **Вращение:** К текущему значению `rotation` добавляется `rotationSpeed`.
    3.  **Сброс:** Если угол поворота превышает 360 градусов, он сбрасывается в 0, чтобы числа оставались небольшими.
*   **Когда вызывается:** Автоматически классом `StarfieldView` внутри цикла анимации (примерно 60 раз в секунду).
*   **Что важно понимать:** Этот метод содержит «математику» анимации. Он ничего не рисует; он просто меняет числа, чтобы при *следующей* отрисовке звезда выглядела иначе.

## 4. Жизненный цикл
*Не применимо (обычный класс).*

## 5. Взаимодействие с интерфейсом (UI)
*   Этот класс не взаимодействует с UI напрямую. Он только хранит данные, которые `StarfieldView` использует для рисования на экране.

## 6. Взаимодействие с другими компонентами
*   **Хранилище данных:** Передает свои свойства (такие как `alpha`, `x`, `y`) классу `StarfieldView` по запросу.

## 7. Общая логика класса
Класс `Star` похож на актера в спектакле. Он знает свое положение, яркость своего костюма и то, как «мерцать». Каждый раз, когда режиссер (`StarfieldView`) говорит «Мотор!» (вызывает `update`), актер немного меняет яркость или слегка поворачивается.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
Представьте одну лампочку на новогодней гирлянде. Этот класс — как память этой лампочки. Он помнит: «Я нахожусь в левом верхнем углу», «Сейчас я горю в полсилы» и «Я медленно вращаюсь».
Каждую секунду лампочка проверяет свои правила: «Должна ли я стать ярче? Да. Хорошо, прибавлю яркости совсем чуть-чуть». Это именно то, что делает метод `update()`.
Класс не «знает», как нарисовать себя на экране; он просто следит за своим собственным состоянием.
