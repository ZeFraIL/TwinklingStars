# Класс: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность)
*   **Назначение:** Это «хозяин» или «менеджер» приложения. Это первый экран, который открывается, когда пользователь нажимает на иконку приложения. Его задача — подготовить окно, в котором будет происходить всё остальное.
*   **Взаимодействие:** 
    *   Загружает файл макета `activity_main.xml`.
    *   Служит родительским контейнером для `StarfieldView`.

## 2. Переменные (Поля класса)
*   *Нет.* Эта конкретная активность очень проста и не хранит собственных переменных данных.

## 3. Методы класса

### Имя метода: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Хранит данные, если приложение было перезапущено (например, после поворота телефона). |
*   **Что делает:** 
    1.  Вызывает родительскую версию `onCreate` (обязательно).
    2.  Устанавливает визуальный макет с помощью `setContentView(R.layout.activity_main)`. Это говорит Android: «Покажи на экране XML-файл, который я разработал».
*   **Когда вызывается:** Ровно один раз при запуске приложения.
*   **Что важно понимать:** Если вы забудете вызвать `setContentView`, пользователь увидит просто пустой белый экран.

## 4. Жизненный цикл (ТОЛЬКО для Activity)
*   **`onCreate()`:** Вызывается при первом создании активности. Здесь мы настраиваем макет.
*   **`onStart()` / `onResume()`:** Явно не прописаны в коде (используется стандартное поведение Android), но они происходят после `onCreate`, чтобы сделать экран видимым и интерактивным.
*   **`onPause()` / `onStop()` / `onDestroy()`:** Обрабатываются системой для экономии заряда батареи, когда пользователь покидает приложение.

## 5. Взаимодействие с интерфейсом (UI)
*   **Макет:** Использует `R.layout.activity_main`.
*   **Компоненты:** Эта активность содержит `StarfieldView` (пользовательский компонент) и `TextView` (заголовок «Twinkling Stars»), определенные в XML.

## 6. Взаимодействие с другими компонентами
*   **Наполнение макета (Inflation):** Читает XML-файл и создает Java-объекты для интерфейса.

## 7. Общая логика класса
1.  Пользователь нажимает на приложение.
2.  Система Android вызывает `MainActivity`.
3.  `MainActivity` говорит: «Покажи макет `activity_main`».
4.  Внутри этого макета `StarfieldView` запускает свою анимацию.

## 8. Упрощенное объяснение
**Объяснение простыми словами:**
`MainActivity` похожа на входную дверь и фундамент дома. Когда вы входите в «дом» (приложение), `MainActivity` открывает дверь и следит за тем, чтобы стены и мебель (интерфейс) были на своих местах. Она сама не выполняет «работу» (как звезды), но предоставляет пространство, в котором эти звезды могут существовать.
